## 預期產物

- `rfp_scan_requirements`（或課程指定 YAML）與 profile／工具的對應說明。
- 至少一條可追溯的**需求 ID → 產物欄位**對照。

## 評分對照

| 等級 | 要點 |
|------|------|
| **及格** | 一條以上需求對到工具與交付物；無空泛「都有掃到」。 |
| **優良** | 能處理需求衝突或優先級（例如時程內無法全掃時的取捨敘述）。 |

## 常見不合格徵兆

- 只列工具名無對應 RFP 條號。
- 與 `configs/profiles.yaml` 實際內容脫節。
