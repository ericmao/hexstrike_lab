## 預期產物

- 目錄結構說明（含 `hexstrike_lab/`、`configs/`、`tests/`）。
- 從 CLI 輸入到最終輸出之資料流（文字或圖）。

## 評分對照

| 等級 | 要點 |
|------|------|
| **及格** | 至少 3 個頂層目錄職責正確；流程不與「手刻 JSON 當報告」混淆。 |
| **優良** | 能指出 `pipeline` 產物目錄與單次 `run` 的差異；圖中有決策或分支。 |

## 常見不合格徵兆

- 只複製 README 首段、無執行路徑。
- 將「掃描器」與「編排器／pipeline」混為一談。
