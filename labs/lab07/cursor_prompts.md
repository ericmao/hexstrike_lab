# Lab 07 — 結構化偵察（角色／任務／輸出格式）

**對照講師手冊：** LAB2（AI 驅動偵察、Nmap、單點深化）。**載體：** hexstrike-ai MCP。

---

## 學習目標

- 使用 **RTF 式**提示（角色、任務、輸出格式）驅動代理選擇 **nmap** 類工作。
- 從 **後端 log 或 MCP 回傳**還原**實際執行的指令列**，完成「指令溯源」。
- 在**單一授權 IP**上進行第二階段（例如 80 埠的目錄／內容發現），範圍寫死於講義。

## 授權與環境

- **僅**課程白名單：例如 `192.168.56.0/24` 或講師指定網段；**嚴禁**改掃其他網段。
- 第二階段目標 IP 須為講師發放之**單一主機**（例如 `192.168.56.101`）。

## 操作步驟

1. 撰寫提示詞（範本意涵，勿照抄未授權 IP）：  
   「角色：資深紅隊偵察員。任務：掃描網段 **&lt;講義填寫&gt;**。要求：列出開放埠、作業系統線索（若工具支援），以**表格**輸出 IP、埠、服務。」
2. 透過 MCP 執行；**儲存**代理與工具輸出。
3. **指令溯源**：在 server log 或工具輸出中找出實際 `nmap`（或其他）參數，複製到實驗紀錄。
4. 針對講義指定之**單一 IP** 的 **80/tcp**，下第二則提示：目錄或敏感路徑發現（僅限該 URL／該主機）。

## 觀察／除錯

- 若代理建議 `-O` 全網段：討論**時間、噪音、授權邊界**是否可接受。
- 比對「模型宣稱執行的」與 log 中「實際執行的」是否一致。

## 驗收標準

- 一份 **Markdown／PDF**：掃描摘要表 + **指令溯源附錄**（至少一條完整 argv）。
- 簡答：LLM **規則式 profile（如 hexstrike_lab `run_when`）** 在何種情境下仍可能選錯工具或參數？

## 講師提醒

- 開源 hexstrike-ai 工具面廣：**本堂僅允許** recon／目錄類；其餘工具若被呼叫應中止並重講範圍。
- 若學員無法存取 server log，改交 **工具原始輸出檔** 作為溯源依據。
