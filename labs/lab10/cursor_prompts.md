# Lab 10 — 程式修補與安全開發（Exploit 腳本現代化）

**對照講師手冊：** LAB5（Python2→3、JSON 輸出、反彈 shell）。**載體：** hexstrike-ai MCP + 本機 Python。

---

## 學習目標

- 將講師提供之**過時／損壞**範例腳本（如 Python 2 語法、錯誤函式庫）**重構為 Python 3**，並將關鍵輸出改為 **JSON**（便於下游稽核）。
- 理解 **AI 產生之「一行式反向連線」**僅在**隔離實驗網段 + 講師監看**下示範。

## 授權與環境

- **必做：** 程式修補 + JSON 輸出 + 單元測試或手動跑通於**本機假資料**。
- **選修（需講師另簽）：** 反彈 shell——listener 位於講師指定 IP（如 `192.168.56.100:4444`），靶機僅限實驗 VM；**禁止**對學員個人手機熱點或公網 IP 實作。

## 操作步驟

1. 取得講師範例（含刻意錯誤）；**勿**先問 AI 全文重寫，先自行試錯。
2. 請 MCP 協助：**僅**回答語法與相容性問題，要求輸出**分段 patch**。
3. 完成：腳本在 Python 3.10+ 可執行；`stdout` 或檔案輸出為**單一 JSON 物件**（欄位由講義定義）。
4. （選）依手冊精神產生「一行式」payload 與 Base64 包裝——**僅在講師按下「執行」後**於隔離環跑 `nc -lvnp 4444`。

## 觀察／除錯

- AI 是否引入**危險** `eval`／`os.system`？要求學員逐行審查。
- 討論：**JSON 輸出**如何支援日後與 `hexstrike_lab` 的 `evidence_record` 合併。

## 驗收標準

- 提交：修補後原始碼、執行截圖、**JSON 範例一筆**、（選）listener 連線截圖與**時間戳**。
- 口試：說明為何「把 exploit 全權交給 AI 改」在實務上不可接受。

## 講師提醒

- 預設**不評分**反彈 shell 成功與否，以**程式品質與安全審查過程**為主。
- 準備**無網路**或**僅內網** VM 映像，避免 payload 誤觸外網。

## 與 hexstrike_lab 對照

- 本 repo 強調 **subprocess 列表參數、無 shell**；學員應對照：若 AI 建議 `shell=True`，必須拒絕並改寫。
