# Lab06 拓樸案例：Kali（Claude Code）— NDR（bridge）— 標的 192.168.1.203

**用途：** 講師備課／學員畫連線圖時對照。**僅限授權實驗室**；IP 請依實際講義置換。

## 邏輯拓樸

```text
┌─────────────────────────────┐         ┌──────────────────┐         ┌─────────────────────┐
│ Kali（Claude Code + MCP）    │         │ NDR（L2 bridge）  │         │ 標的主機             │
│                             │         │ 透明橋接，同廣播域 │         │ 192.168.1.203       │
│ • hexstrike_server :8888    │  L2/L3  │ 可見流量、不終結   │         │ （Lab 白名單目標）    │
│ • Ollama :11434（選）       │ ──────► │ TCP 連線           │ ──────► │ 僅在核准任務中使用   │
│ • hexstrike_mcp（stdio）    │         │                    │         │                     │
└─────────────────────────────┘         └──────────────────┘         └─────────────────────┘
```

## 與 MCP 設定的關係（重要）

| 項目 | 典型設定 | 說明 |
|------|-----------|------|
| **`--server`（MCP → hexstrike HTTP）** | `http://127.0.0.1:8888` | 後端跑在 **Kali 本機** 時，與 NDR／標的 IP **無關**。 |
| **標的 IP `192.168.1.203`** | 寫在 **RoE／講義／對話提示**，**不是** MCP JSON | Lab07 起掃描／工具參數用；須符合白名單。 |
| **NDR bridge mode** | 一般不改變 IP 路由語意 | 同子網可直達標的；NDR 側可見 East-West 流量供偵測／錄影。 |

若 `hexstrike_server` 跑在**另一台**主機，則 `--server` 改為該主機 IP:port（需防火牆允許）。

## 驗收圖上建議標註

1. Claude Code（MCP stdio）→ 本機 `hexstrike_mcp.py` → HTTP → `127.0.0.1:8888`。
2. （選）hexstrike_server → Ollama `127.0.0.1:11434`。
3. 另用虛線或不同顏色標：**授權標的** `192.168.1.203` 與 **NDR** 在資料路徑上的位置（與 MCP 控制平面分開）。
